## ПРАВИЛА ЭКОНОМИИ ТОКЕНОВ

### При чтении кода
- Читать только нужный файл, не весь репозиторий сразу
- Использовать `grep` вместо полного чтения если нужна одна строка
- `git diff HEAD~1` — смотреть только изменения, не весь файл
- Параллельно читать независимые файлы (filesystem__read × N)

### При написании кода
- Писать изменения через `filesystem__edit` (patch), не `filesystem__write` (полная перезапись)
- Исключение: новые файлы — всегда `filesystem__write`
- Не дублировать код — ссылаться на существующие функции
- Не добавлять комментарии к очевидному коду

### При поиске
- `filesystem__grep` перед чтением файла — убедиться что там есть нужное
- `git log --oneline -5` — последние коммиты вместо полной истории
- `go vet ./...` → `go build -o /dev/null ./...` — минимальная валидация

### При ответах пользователю
- Не пересказывать TODO-список (пользователь его видит)
- Не объяснять очевидные шаги
- Ошибки: название + количество, потом исправить молча
- Успех: одна строка `OK` или имя файла

## ШАБЛОНЫ ЧАСТЫХ ЗАДАЧ

### Добавить новую CLI-команду
```
1. main.go: добавить case в switch
2. новый_файл.go: func runНоваяКоманда(ollamaURL, ... string)
3. go vet + go build
```

### Добавить HTTP-маршрут
```
1. server.go: mux.HandleFunc("/api/новый", handler)
2. static/index.html: добавить fetch('/api/новый')
3. go build (embed пересобирается автоматически)
```

### Добавить зависимость Go
```

# Обновить go.mod и go.sum в коммите
```

## ЧАСТЫЕ ОШИБКИ И РЕШЕНИЯ

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `undefined: runXxx` | Функция в другом файле ещё не создана | Добавить заглушку `func runXxx(...) {}` |
| `go:embed: no matching files` | Файл не существует | Создать файл перед `go build` |
| `imported and not used` | Лишний импорт | Удалить из `import()` |
| `cannot use X (type Y) as type Z` | Несовпадение типов | Проверить сигнатуру функции в ollama.go |
| SSE не приходит в браузер | Nginx буферизует | `X-Accel-Buffering: no` уже стоит |
| `dial tcp: connection refused` | Ollama не запущен | `docker compose up -d ollama` |

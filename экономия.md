## ТОКЕН-ЭКОНОМИЯ В КОДЕ

```
compress.go:
  maxHistoryMessages = 24   // сжимаем если больше
  keepRecentMessages = 8    // оставляем последних
  agentTokenBudget   = 3500 // лимит для агента
  maxToolOutput      = 2048 // обрезка вывода инструмента

agent.go:
  maxAgentSteps = 8         // не более N итераций
  temp агента = 0.4         // низкая = меньше токенов на рассуждение

rag.go:
  chunkText(text, 400, 60)  // ~400 слов, 60 overlap
  topK = 4, порог = 0.3     // только релевантные чанки
  embedBatchParallel = 4    // параллельное эмбеддирование
```

## ШАБЛОНЫ ЗАДАЧ

```
Добавить инструмент агента:
  tools.go → AllTools["name"] = &ToolDef{..., Run: func(args) (string,error)}
  (агент подхватит автоматически через ToolsPrompt())

Добавить API-маршрут:
  server.go → mux.Handle("/api/новый", protected(handler))
  index.html → fetch('/api/новый')

Добавить защищённый маршрут:
  server.go → mux.Handle("/api/...", protected(handler))  // с auth
  server.go → mux.Handle("/api/...", adminOnly(handler))  // admin only

Изменить RAG-параметры:
  rag.go: chunkText(text, 400, 60) → (targetWords, overlap)
  server.go: rag.Search(ctx, msg, 4) → topK

Изменить лимиты контекста:
  compress.go: maxHistoryMessages / keepRecentMessages / agentTokenBudget

Добавить поле в конфиг:
  config.go: AppConfig → новое поле, LoadConfig → парсинг, MergeEnv → env переменная
  main.go: передать в runServer
```

## ЧАСТЫЕ ОШИБКИ

```
duplicate func name    → переименовать одну версию
undefined: X           → stub или создать файл
go:embed missing       → создать файл до go build
SSE не стримит         → X-Accel-Buffering: no (уже стоит в server.go)
RAG пустой ответ       → порог similarity 0.3, нет nomic-embed-text → fallback tfidf
агент зациклился       → maxAgentSteps=8 защищает
config parse error     → двоеточие в значении → взять в кавычки "http://..."
```
